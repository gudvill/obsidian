JSON (JavaScript Object Notation) - это текстовый формат обмена данными, который используется для хранения данных и их передачи между различными системами и приложениями. Cоздан на основе синтаксиса JavaScript, используется во многих других языках (Python, Ruby, Java, C# и др.).  
  
В формате JSON данные организованы в виде пар **ключ - значение**. Основной принцип JSON - это упрощение структуры данных, что делает его более понятным и лёгким в использовании.  
  
**Структура JSON**:  
- ключи всегда записываются как строки в двойных кавычках;
- значения могут быть разного типа: строки, числа, булевы значения, объекты, массивы или null;
- каждый элемент разделяется запятой, а объект или массив начинаются и заканчиваются с фигурных или квадратных скобок соответственно.

Пример (массив объектов):
`[`  
	`{`  
		`"name": "Alice",`  
		`"age": 30,`  
		`"isEmployed": true`  
	`},`  
	`{`  
		`"name": "Bob",`  
		`"age": 25,`  
		`"isEmployed": false`  
	`}`  
`]`
### Преимущества JSON перед другими форматами

1. читаемость и простота (в сравнении с XML, где присутствуют теги);
2. малый объём данных (в сравнении с XML, где присутствуют теги);
3. гибкость в представлении данных (CSV работает только с табличными данными);
4. лёгкость интеграции с ЯП (структура аналогична объектам и массивам в JavaScript);
5. широкая поддержка в веб-браузерах, API и серверах.
### Работа с JSON

- **JavaScript**:
	JSON.parse() - преобразование JSON-строки в объект;
	JSON.stringify() - преобразование объекта в JSON-строку.
 - **Python**:
	data = json.loads(json_string) - для чтения JSON;  
	json_string = json.dumps(data) - для записи объекта в JSON.  

**Парсинг JSON** - преобразование данных, представленных в формате JSON, в объекты, которые можно использовать в коде программы. 

**Сериализация** - это процесс преобразования объектов или структур данных в строку JSON, чтобы передать или сохранить её. Используется при передаче данных через веб-протоколы, такие как HTTP, между клиентом и сервером.

Одной из основных сфер применения JSON является обмен данными через [[API]]. Веб-приложения активно используют JSON для отправки запросов и получения ответов. При работе с REST API данные в запросе или ответе часто передаются в формате JSON. Клиент отправляет HTTP-запрос, который содержит JSON-данные, а сервер возвращает ответ также в формате JSON. Это позволяет обеспечить унифицированное взаимодействие между клиентами и серверами.
### Примеры формата данных JSON
**1. простой объект** - объект с несколькими ключами и значениями (объект в {});
**2. массив объектов** - несколько объектов (объекты в [] каждый объект в {});
**3. вложенные объекты** - объекты, или списки, внутри других объектов (в значении у какой-то пары ключ-значение (у атрибута)).
